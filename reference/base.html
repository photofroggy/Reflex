3. base module¶

The base module contains different base classes for different components of the event system.

3.1. Ruleset - Binding management¶

class reflex.base.Ruleset(args, mapref, stdout, stddebug)¶

Default Ruleset.

Input parameters:

  • args - Any arguments passed to the event manager object when the instance is created.
  • dict mapref - A reference to the dictionary storing all of the event bindings stored in the system.
  • callable stdout - A reference to the method being used to write output to the screen. This is given by the event manager object.
  • callable stddebug - A reference to the method being used to write debug messages to the screen. This is given by the event manager object.

The Ruleset object provides an interface for managing bindings and events on a per-event basis. One Ruleset is assigned to one type of event only.

An instance of this class is used for all events that do not have a specific ruleset assigned to them.

bind(meth, event, **options)¶

Create an event binding.

This is the method called by the event manager object’s own bind() method. It ensures that there are no clashes between the given binding details and any existing event bindings.

If there is a clash, then the method returns None, otherwise an instance of the Binding class is returned. The instance is also added to the object referenced by self.mapref.

init(*args)¶

init method.

This method is called by the __init__() method. Subclasses should overwrite this method if they need to do anything on initialisation.

This is a do-nothing method by default.

run(binding, data, *args)¶

Run a given event binding.

Input parameters:

  • binding - An instance of the Binding class.
  • data - An instance of the Event class. This represents the event being triggered.
  • rules - This is a list of the items related to the event. Typically, these items are compared to the binding’s own options list. Custom Rulesets may use both lists in their own way, if appropriate.
  • args - Additional arguments given to the event manager’s trigger() method.
set_map(mapref)¶

Set the reference for the event map.

Input parameters:

  • dict mapref - As in the __init__() method, this parameter is a reference to the dictionary storing all of the event bindings being stored in the event system.

This method should only ever be called by the event manager object’s clear_bindings() method.

unbind(meth, event, **options)¶

Remove an event binding.

Similar to the bind() method of this class, except the binding is removed rather than added. This method True if a matching event binding was found and removed, False if no bindings were removed.

3.2. Reactor - Base for reactive classes¶

class reflex.base.Reactor(manager, *args, **kwargs)¶

Reactor base class.

Input parameters:

  • manager - This is a reference to an instance of a reflex.control.EventManager object. This is used here to provide the base reactor class with the bind(), unbind(), handler() and trigger methods. For more information about these methods, see the documentation for the reflex.control.EventManager class.
  • args and kwargs - Any additional parameters given here are passed to the init() method.

In the reflex system, a reactor is an object containing several methods which react to events, based on certain parameters. The methods that react to events are usually referred to as ‘event handlers’.

To define a method as an event handler, the method must be bound to an event using the bind() method. This creates an event binding. An event binding is an object (instance of reflex.data.Binding) that holds information about the relationship between a method and an event.

Developers adapting Reflex for their own applications are advised to use the reactor class as a base for any base classes they might want to create.

To get an idea of how to properly create and use reactors, refer to the tutorials.

bind(method, event, **options)¶

This is a wrapper for reflex.control.EventManager.bind().

handler(event, **options)¶

Can be used as a decorator to bind events.

init(*args, **kwargs)¶

Reactor configuration.

This is a do-nothing method. Subclasses should extend this method and use it to create event bindings.

unbind(method, event, **options)¶

Another wrapper like bind.
