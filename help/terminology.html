TerminologyÂ¶

This documentation website and the reflex package both use certain terminology to describe the concepts and ideas behind the package, and the functionality of the package. This page attempts to explain the meaning of the terms and the reasoning behind using those terms.

Package Battery
The term “battery” can refer to a collection of related items that are grouped together[1]. In Reflex, a “package battery” refers to the nature of the Battery classes. Battery objects essentially represent a given package as a battery of modules and specific objects.
Reactor
A Reactor is a collection of event handlers. An event handler responds to, or reacts to, a given event. Because a Reactor is a collection of event handlers, the Reactor itself should be seen as reacting to any number of events in any number of ways.
Reflex
The name of the package itself. A Reflex is an instinctive reaction to a stimulus of some sort[2]. The reflex package provides an event system that allows applications to react to events. The name of package comes from the idea of reflex reactions.
Reflexive
A system can be referred to as a reflecive system if it uses the reflex package or implements a Reflex-like event system.
Ruleset
A ruleset is a self-contained object or class that modifies the behaviour of an event system during certain event-related operations.
